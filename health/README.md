# health — превентивный чекап с Cursor

Рабочая папка для ассистента `health-checkup`: хранит анализы, анамнез и краткую сводку, чтобы Cursor-агент мог разбирать результаты с уклоном в доказательную превентивную медицину и longevity.

**Это не медицинская помощь.** Решения принимает врач очно. Агент — инструмент для подготовки, систематизации и вопросов к специалисту.

## Структура

```
health/
├── .cursor/skills/health-checkup/SKILL.md   инструкции для агента
├── profile.md                               анамнез (заполняете вы)
├── summary.md                               роллинг-саммари (ведёт агент)
├── labs/                                    PDF / сканы / фото анализов
└── .gitignore                               прячет медданные от git
```

## Как начать

1. Откройте **корневую папку проекта** в Cursor: **File → Open Folder → `ai-skills-example-public`**. Скилл `health-checkup` активируется автоматически.
2. Заполните `profile.md`. Что не знаете — оставьте «не указано», не выдумывайте.
3. Сложите файлы анализов в `labs/`. Русский, английский, немецкий — ок. PDF лучше сканов, сканы лучше фото.
4. Напишите агенту: «разбери анализы», «пройди со мной чекап», «сравни новые результаты с прошлыми» или спросите про конкретный маркер.

Агент при первом запуске может задать вопросы, чтобы обновить анамнез, и попросить переименовать файлы, если они названы непонятно.

## Именование файлов в `labs/`

Формат: `YYYY-MM-DD_тип_лаборатория.расширение`

Примеры:

- `2026-02-14_CBC_Invitro.pdf` — общий анализ крови.
- `2025-11-03_lipids_Helix.pdf` — липидный профиль.
- `2026-01-20_thyroid_Synevo.pdf` — щитовидка.
- `2025-09-12_ultrasound-thyroid_MedSI.pdf` — УЗИ щитовидной железы.

Дата — **когда сдали** (а не когда получили результат). Это нужно, чтобы агент строил корректную динамику.

## Приватность

- `.gitignore` исключает `labs/`, `profile.md` и `summary.md` из git. Коммит анализов случайно не произойдёт.
- Облачные бэкапы (iCloud, Dropbox, Time Machine) `.gitignore` **не** контролирует. Если это критично — держите папку вне синкаемых директорий.
- Не кладите в `profile.md` номера полисов, паспортов и страховок — агенту они не нужны.

## Если что-то не распозналось

- Рукописные заключения и мутные фото модель читает плохо. Для критичных цифр — перепроверяйте глазами или наберите вручную в отдельный `.md` рядом с файлом.
- Если агент говорит «не читается» — пришлите скан получше или продиктуйте значения в чат.

## Что агент делает и чего не делает

Делает: систематизирует маркеры, строит динамику, подсвечивает отклонения по референсу лаборатории и по longevity-ориентирам, выделяет красные флаги, предлагает анализы для добора и вопросы к врачу.

Не делает: не ставит диагнозы, не назначает рецептурные препараты и дозы, не заменяет очный приём, не использует цифры «из головы» — только то, что есть в ваших файлах и `profile.md`.
